# Linux Troubleshooting Lab

Bem-vindo ao **Linux Troubleshooting Lab**.

Este repositório tem como objetivo aprofundar os conhecimentos em **Linux**, com foco em **diagnóstico e resolução de problemas reais** em ambientes de infraestrutura.

O projeto foi criado como parte de um plano de desenvolvimento técnico para consolidar fundamentos avançados do sistema operacional, utilizando uma abordagem **prática e orientada a cenários reais**.

---

## Objetivos do Projeto

- Desenvolver habilidade de troubleshooting em sistemas Linux
- Aprimorar o uso de ferramentas nativas do sistema operacional
- Entender o comportamento de serviços, processos e rede
- Criar raciocínio técnico para identificação de falhas
- Documentar incidentes e soluções de forma profissional

---

## Conhecimentos Trabalhados

- Gerenciamento de processos e recursos
- systemd e gerenciamento de serviços
- Logs e diagnóstico (`journalctl`, `rsyslog`)
- Permissões, usuários e hierarquia de diretórios
- Redes:
  - IP e rotas
  - Portas e sockets (`ss`, `netstat`)
  - DNS e conectividade
- Firewall e controle de acesso
- Troubleshooting orientado a logs

---

## Estrutura do Laboratório

O laboratório é composto por **cenários de falhas simuladas**, onde cada cenário descreve:

1. O problema identificado
2. O processo de investigação
3. As ferramentas utilizadas
4. A solução aplicada
5. Considerações finais

Cada cenário está documentado para reproduzir situações comuns encontradas em ambientes de produção.

---

## Cenários Implementados

- Serviço não inicia corretamente
- Serviço em execução, porém inacessível externamente
- Problemas de resolução DNS
- Porta bloqueada por firewall
- Falha causada por permissões incorretas

*(os cenários serão adicionados progressivamente)*

---

## Ambiente Utilizado

- Sistema Operacional: Linux (Ubuntu 22.04)
- Acesso via SSH
- Ferramentas nativas do sistema
